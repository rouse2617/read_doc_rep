# 使用技巧


我们存放数据是可以用 key-value的形式来存，现在我们使用最简单的方式存  

```
echo "{key : value  }" > db.file
```



这是一种很简单的方法，后续续集存的话使用  >> 追加即可

```
echo "{key1 : value1  }" >> db.file
```



我们要查找的话，通过key来找，我们直接用grep 过滤出来即可

```
grep "key" db.file
```



以上是较为简单的一种数据存储实现方式，优势在于直接在尾部追加写，有着非常好的性能（顺序IO），这里和日志一样；但这里有几个问题

-   查找效率
-   修改已存在的kv值 
-   文件增长
-   kv 信息丢失
